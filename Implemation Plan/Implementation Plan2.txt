# [Goal Description]

사용자 로그인 시스템을 도입하고, 로그인한 사용자별로 이미지 생성 토큰(크레딧)을 관리하는 방안에 대한 수행 계획입니다.
사용자 요청에 따라 **[ Option A (Supabase Auth) ] + [ Option 2 (Client-Side + Local DB Caching) ]** 조합으로 구축됩니다.

## Proposed Strategy: Supabase Auth + Local Token Caching

현재 Vite + React (순수 프론트엔드) 구조의 이점을 살리면서 가장 빠르고 효율적으로 인증 및 토큰 제한을 거는 방법입니다.

### 1. 사용자 인증 (Supabase Auth)
- **Supabase SDK 연동**: `App.tsx` 최상단 혹은 전역 상태(Context/Zustand)에 Supabase Auth Provider를 탑재합니다.
- **로그인/회원가입 모달 구현**: 이메일/비밀번호 방식과 구글 소셜 로그인 등을 쉽게 띄울 수 있는 UI 컴포넌트를 제작합니다.
- **라우팅 보호**: 로그인되지 않은 사용자는 기능 버튼(GENERATE 등)을 누를 수 없도록 막거나, 진입 시 바로 로그인 모달을 띄웁니다.

### 2. 토큰 관리 (로컬 스토리지 캐싱)
- **원리**: 백엔드 DB를 무겁게 구축하지 않고, 프론트엔드 단에서 토큰을 계산합니다.
- **초기 토큰 부여**: 사용자가 성공적으로 로그인하면, 브라우저의 `localStorage` (또는 유저별 로컬 기록)에 초기 토큰(예: 5개)을 세팅합니다.
- **차감 프로세스**: 
  1. Generate 버튼 탭
  2. 로컬 스토리지의 토큰 잔량 확인 (0원이면 동작 거부 및 결제/충전 안내 모달)
  3. API 호출 성공 시 로컬 스토리지의 토큰 카운트를 -1 차감
- **한계점 (인지 사항)**: 
  - 로컬 스토리지 기반이므로, 브라우저의 개발자 도구를 열어 캐시를 조작하거나 앱 데이터를 삭제하면 토큰이 초기화될 위험이 있습니다.
  - 현재 단계(알파 테스트 또는 무료 데모 배포)에서는 서버 비용과 시간을 크게 절약할 수 있어 매우 적합합니다. 향후 정식 과금 모델로 전환 시에만 Option 1(서버 DB 직접 제어)로 업그레이드하면 됩니다.

### [NEW] 2-1. 관리자(Admin) 모드 특권
- **원리**: 로그인된 사용자의 이메일 계정을 판별하여, 특정 계정(예: 지정된 관리자 이메일 목록)인 경우 무제한 생성 권한을 부여합니다.
- **로직**:
  1. 로그인 세션 확인 시 사용자 이메일(`user.email`)이 사전에 하드코딩된 관리자 이메일 목록(또는 `env` 환경 변수에 등록된 목록)에 포함되는지 검사합니다.
  2. 관리자 계정으로 판별된 경우, UI 상에 'Token: ∞ (Admin)' 형태로 표시되며 잔여 토큰 체크 및 차감 로직을 전면 우회(Bypass)합니다.
  3. 일반 일반 유저와 완전히 동일한 경험을 하되, 생성 버튼을 무한정 누를 수 있게 됩니다.

### 3. Upscaler (업스케일링 서버 관련)
- 현재 프로젝트에서는 **Stability AI API (v2beta/stable-image/upscale/conservative)** 가 연결되어 있습니다.
- Google AI Studio를 사용할 경우, 자체적인 고해상도 이미지 처리 능력을 적용할 수 있으나, 전용 '업스케일링 특화 모델' 여부는 구글 정책 및 제공 API (Imagen 3 등)에 따라 조율해야 합니다.

## Verification Plan

### Manual Verification
1. **로그인 기능 테스트**: 이메일 회원가입 및 구글 소셜 로그인 버튼을 눌러 성공적으로 세션이 생성되는지 확인합니다.
2. **토큰 차감 테스트**: 
   - 가입 직후 초기 토큰(예: 5/5)이 UI 타이틀 쪽에 표시되는지 확인.
   - 생성을 1회 진행할 때마다 4, 3, 2로 정상 차감되는지 여부 점검.
3. **토큰 고갈 테스트**: 0이 되었을 때 GENERATE 버튼이 막히고 경고 모달이 뜨는지 확인.
4. **로컬 캐시 보존**: 새로고침(F5)을 하더라도 남은 토큰 갯수가 유지되는지 확인.
